55 /*-------------------------------------------------------
56 * Configuration Options
57 *-------------------------------------------------------
58 * All options described below should be passed as a compiler flag to all
59 files using Unity. If you must add #defines, place them BEFORE the #include
60 above.
61
62 * Integers/longs/pointers
63 * - Unity attempts to automatically discover your integer sizes
64 * - define UNITY_EXCLUDE_STDINT_H to stop attempting to look in
65 <stdint.h>
66 * - define UNITY_EXCLUDE_LIMITS_H to stop attempting to look in
67 <limits.h>
68 * - If you cannot use the automatic methods above, you can force Unity
69 by using these options:
70 * - define UNITY_SUPPORT_64
71 * - set UNITY_INT_WIDTH
72 * - set UNITY_LONG_WIDTH
73 * - set UNITY_POINTER_WIDTH
74
75 * Floats
76 * - define UNITY_EXCLUDE_FLOAT to disallow floating point comparisons
77 * - define UNITY_FLOAT_PRECISION to specify the precision to use when
78 doing TEST_ASSERT_EQUAL_FLOAT
79 * - define UNITY_FLOAT_TYPE to specify doubles instead of single
80 precision floats
81 * - define UNITY_INCLUDE_DOUBLE to allow double floating point
82 comparisons
83 * - define UNITY_EXCLUDE_DOUBLE to disallow double floating point
84 comparisons (default)
85 * - define UNITY_DOUBLE_PRECISION to specify the precision to use when
86 doing TEST_ASSERT_EQUAL_DOUBLE
87 * - define UNITY_DOUBLE_TYPE to specify something other than double
88 * - define UNITY_EXCLUDE_FLOAT_PRINT to trim binary size, won't print
89 floating point values in errors
90
91 * Output
92 * - by default, Unity prints to standard out with putchar. define
93 UNITY_OUTPUT_CHAR(a) with a different function if desired
94 * - define UNITY_DIFFERENTIATE_FINAL_FAIL to print FAILED (vs. FAIL) at
95 test end summary - for automated search for failure
96
97 * Optimization
98 * - by default, line numbers are stored in unsigned shorts. Define
99 UNITY_LINE_TYPE with a different type if your files are huge
100 * - by default, test and failure counters are unsigned shorts. Define
101 UNITY_COUNTER_TYPE with a different type if you want to save space or have
102 more than 65535 Tests.
103
104 * Test Cases
105 * - define UNITY_SUPPORT_TEST_CASES to include the TEST_CASE macro,
106 though really it's mostly about the runner generator script
107
108 * Parameterized Tests
109 * - you'll want to create a define of TEST_CASE(...), TEST_RANGE(...)
110 and/or TEST_MATRIX(...) which basically evaluates to nothing
111
112 * Tests with Arguments
113 * - you'll want to define UNITY_USE_COMMAND_LINE_ARGS if you have the
114 test runner passing arguments to Unity
